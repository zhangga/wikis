## 优化摘要

Claude Code 和 Bun 创始人的结对编程直播，真正值得看的不是“AI 又会写代码了”，而是 AI 开始接住 issue、测试、PR、review 这些真实工程环节。本文拆解 AI 工程流正在如何重塑开发协作，并给普通团队一套五项自检和可直接照抄的提示词模板。

## 备选摘要

1. AI 编程的下一阶段，不是生成更多代码，而是把一个模糊 issue 推进成可验证、可评审、可合并的 PR。这篇文章从 Claude Code 与 Bun 的直播出发，拆解程序员能力迁移的方向。
2. Bun 官方提到，Claude Code bot 已经在 Bun 仓库里大量修 bug、开带测试的 PR、回应 review comments。这个信号说明：AI 正从代码助手变成工程协作者。
3. 如果你的团队想把 AI 编程真正用起来，别先追求全自动。先检查 issue、测试、仓库约定、review 分工、发布回滚这五块能不能接住 AI 的速度。
