# Hermes Agent / OpenClaw 公众号选题设计

## 目标

面向普通内容创作者，写一篇不是教程、而是趋势预警的公众号文章。核心问题不是“哪款 Agent 参数更强”，而是：

1. Hermes Agent 最近为什么突然热起来。
2. 它和 OpenClaw 代表的是哪两条不同的产品路线。
3. 这股热度对普通创作者的工作流意味着什么。

## 目标读者

- 对 AI Agent 感兴趣，但不是重度开发者的内容创作者
- 已经听说过 OpenClaw，或最近频繁刷到 Hermes Agent
- 更关心“该不该关注”和“什么时候该上手”，而不是底层架构细节

## 文章骨架

采用 `现象拆解型`：

1. 现象是什么：最近 Hermes Agent 相关文章、迁移讨论、对比内容密集出现。
2. 为什么会火：不仅是功能，而是“会长大的 Agent”这套叙事更容易被普通人理解和转述。
3. 和 OpenClaw 的深度差异：不是简单替代，而是两种产品阶段与两种用户心智。
4. 这说明了什么：Agent 产品的竞争，开始从“能不能跑”转向“能不能进入日常工作流并持续积累”。
5. 创作者该怎么判断：给出观望、试用、继续留在原工具的判断清单。

## 比较维度

文章重点比较以下 5 个维度：

1. 产品叙事：成长型 Agent vs 平台型个人助手
2. 核心能力：技能、记忆、异步任务、前端入口
3. 交互入口：CLI / WebUI / API vs 多渠道消息入口 / Canvas / Gateway
4. 运维门槛：普通人能否快速讲清并开始使用
5. 适用场景：单人创作工作流 vs 多渠道常驻助手

## 拟定主判断

Hermes Agent 这波火起来，真正值得注意的，不是它已经在体量上超过 OpenClaw，而是它把 Agent 的卖点从“功能很多”改写成了“会自己长大”。这对普通创作者更有吸引力，也意味着 Agent 赛道开始从极客玩具走向工作流产品。
