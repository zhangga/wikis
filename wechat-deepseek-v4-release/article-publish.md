## 优化摘要

DeepSeek V4 已经进入 API 与开源权重阶段。相比 1.6T 参数、1M 上下文这些显眼数字，更值得看的，是它把低价 API、Flash/Pro 分层、MIT 开源和长上下文 Agent 工作流绑在了一起。这篇文章用 3 张数据图拆清楚它真正改变的地方，以及接下来该看哪 5 个信号。

## 备选摘要

DeepSeek V4 发布后，最容易被传播的是参数和跑分。但这次真正重要的，是开源模型开始用 1M 上下文、低价 API 和双模型分工，争夺开发者的主工作流。今天的 V4 不是终点，而是一次行业压力测试。
