# AGENTS.md

本仓库用于把日常随笔整理为多平台内容（微信公众号 / 小红书）。

## 1. 范围与硬约束

- 所有产出必须保存在**当前仓库根目录**及其子目录。
- 禁止写入其他仓库或外部博客目录（如 `source/_posts`）。
- 新主题必须放在独立子目录，不在根目录散落稿件。
- `research/` 目录用于平台研究、爆款拆解、写作 Playbook，不放发布稿件。

## 2. 目录结构规则

顶层目录约定：

| 目录 | 用途 |
|---|---|
| `research/` | 选题研究、平台数据分析、写作 Playbook |
| `wechat-<topic>/` | 微信公众号稿件 |
| `xhs-<topic>/` | 小红书稿件 |

主题目录只约束前缀：

- 微信：`wechat-<topic>/`
- 小红书：`xhs-<topic>/`

各平台最小结构：

| 平台 | 必需文件/目录 | 可选文件 |
|---|---|---|
| 微信公众号 | `article.md`、`imgs/` | `article-viral.md`、`article-publish.md`、`article-wechat.html` |
| 小红书 | `note.md`、`caption.md`、`cards.md`、`imgs/` | 无 |

## 3. 内容与配图规范

微信公众号（`article.md`）：

- 中文写作，保持当前仓库既有文风（开篇观点、分节展开、结尾行动建议）。
- 默认不使用 frontmatter（除非用户明确要求）。
- 图片仅使用相对路径：`imgs/<filename>.png` 或 `imgs/<filename>.svg`。
- 生成公众号内容时，默认同时生成可用于公众号发布的封面图，保存在对应主题目录 `imgs/` 下。
- 发布到公众号前，默认需要产出并使用优化后的摘要，不直接沿用正文首段替代。
- 默认先在以下四类骨架中选 1 类再开写：`热点快评型`、`现象拆解型`、`人物故事型`、`方法复盘型`。
- 标题默认至少先产出 5 个候选，再选 1 个主标题；标题优先给出判断、代入感或结构承诺，而不是只描述主题。
- 开头前 3 段默认完成三件事：把读者带进场景、直接下判断、告诉读者读完能带走什么。
- 正文默认不是顺着铺陈，而是按小节推进；每个小节至少要有一个明确判断句，不写流水账式长段铺垫。
- 每篇公众号文章默认至少包含两项传播资产：一句可被转述的判断，一个可被带走的动作或框架。

小红书：

- `note.md`：完整笔记正文。
- `caption.md`：发布文案（标题、互动引导、标签建议）。
- `cards.md`：分卡文案，与 `imgs/` 图片编号一一对应。
- 默认按“小红书原生表达”写，不沿用公众号式铺垫。优先采用以下三类结构之一：`结果清单型`、`认知反差型`、`情绪入口 + 方法型`。
- 标题默认至少先产出 5 个候选，再选 1 个主标题；标题优先满足“先让人停下”而不是“讲完整”。
- 封面与前 2 页默认承担“钩子 + 结论”职责，不把背景铺垫放在最前面。
- `cards.md` 默认要求每页只讲 1 个点，每页都能独立截图传播，不写成长段文章切片。
- 小红书内容默认必须包含可收藏的明确结构，如：步骤、清单、对照、错误、动作、模板、公式、阶段之一。
- 小红书正文与卡片默认要有“我试过 / 我踩过坑 / 我最后留下什么”的证据感，避免只有抽象感悟。
- 互动引导默认不问过宽的问题，优先用二选一、补充选择、站队、追问成本/效果/卡点的方式提高评论率。

配图：

- 全部放在对应主题目录 `imgs/` 下。
- 优先提供 `.png`；如需可编辑源文件可同时保留 `.svg`。
- 文件名使用数字前缀保证顺序（如 `01-...`、`02-...`）。

## 4. 交付检查（必须）

完成前逐项检查：

1. 文件是否都在当前仓库内。
2. 目录是否符合 `wechat-` / `xhs-` 前缀规则。
3. 是否满足对应平台最小文件结构。
4. 文内图片相对路径是否全部有效。
5. 微信公众号内容如需发布，是否已准备封面图并完成摘要优化。
6. 微信公众号内容是否满足以下质量门槛：标题不平、开头快速进入判断、正文按骨架推进、文末有明确落点。
7. 小红书内容是否满足以下质量门槛：标题有钩子、首屏先给结论、每页单点成立、存在明确收藏点、互动问题具体。
8. 是否清理了临时稿、重复稿、空目录。
