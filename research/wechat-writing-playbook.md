# 公众号写作 Playbook

用途：以后每次写公众号内容，默认先走这份流程，再产出 `wechat-<topic>/` 下的正式稿件。

关联研究：
- [公众号爆款文章拆解](/C:/work/github/wikis/research/wechat-quality-study-2026-04-07.md)

## 1. 先选文章骨架

每次开写前，先明确这篇更接近哪一种，不混着写。

### 1. 热点快评型

适合：
- 新产品发布
- 行业大事件
- 公共议题的第一时间判断

默认结构：
- 事件是什么
- 为什么值得重视
- 我们应该怎么理解

### 2. 现象拆解型

适合：
- 新平台玩法
- 爆红现象
- 某个内容或产品为什么突然火

默认结构：
- 现象是什么
- 为什么会火
- 这说明了什么
- 后面会走向哪里

### 3. 人物故事型

适合：
- 分享嘉宾、作者、创始人的路径
- 非虚构选题
- 用具体人物折射更大的问题

默认结构：
- 先把人物带出来
- 再讲关键转折
- 最后提炼成更大的判断

### 4. 方法复盘型

适合：
- 听完分享后的方法沉淀
- 工作流、创作法、成长路径总结
- 想让读者读完就能行动

默认结构：
- 先给核心结论
- 再拆 3 到 5 个关键方法
- 结尾给行动建议

## 2. 标题生成规则

每次至少先写 5 个标题候选，再选主标题。

标题默认检查 5 件事：

1. 有没有判断，而不是只有主题
2. 有没有代入感，而不是纯概念
3. 有没有冲突、反差或意外
4. 有没有结构承诺，例如“3个判断”“4个动作”“一条路径”
5. 有没有“现在就值得点开”的理由

如果标题只是把主题换个说法，通常不够。

## 3. 开头三段的固定职责

### 第 1 段：把读者带进场景

不要先讲长背景，先讲：

- 今天发生了什么
- 我为什么会想写这篇
- 这件事刺中了什么问题

### 第 2 段：直接下判断

这一段要回答：

- 这件事为什么重要
- 我的核心观点是什么

### 第 3 段：告诉读者读完能带走什么

这一段常见写法：

- 我这篇不打算重复新闻，我更想拆清楚什么
- 我会从 3 个角度讲透这件事
- 如果你也在做 X，这篇最值得看的是什么

## 4. 正文节奏怎么控

公众号可以长，但默认每一节都要完成推进。

建议每个小节都包含：

1. 一个小标题
2. 一句本节判断
3. 一段事实、案例或解释
4. 一句把意义抬高一层的收束

换句话说，不写“散着顺下来”的正文，而写“每节都有结论”的正文。

## 5. 四种最稳的正文推进方式

### 方式 A：判断 -> 证据 -> 推演

适合快评和行业观察。

### 方式 B：现象 -> 机制 -> 影响

适合平台玩法、产品趋势、流量分析。

### 方式 C：故事 -> 转折 -> 提炼

适合人物型和分享改写。

### 方式 D：问题 -> 拆解 -> 动作

适合方法论、复盘、经验沉淀。

## 6. 公众号最需要的两种“传播资产”

每篇文章默认至少准备这两样：

### 1. 一句可被转述的判断

例如：
- 真正重要的不是 X，而是 Y
- 这次变化的关键不在功能，而在门槛被改写了

### 2. 一个可被带走的动作

例如：
- 读完后可以照着做的 3 个动作
- 一张判断清单
- 一个简单框架

没有这两样，文章就容易停留在“读完觉得不错”，而不是“愿意转发/收藏”。

## 7. 结尾默认怎么收

默认不要只说“以上，就是我的一些思考”。

结尾更推荐做三件事之一：

1. 回收全文观点
2. 给读者一个行动建议
3. 把讨论抬到更大的问题上

如果主题适合，也可以顺手留一个具体问题，方便后续做评论区互动。

## 8. 摘要优化规则

`article-publish.md` 里的摘要，默认不要直接复制正文首段。

好摘要通常做到三件事：

1. 交代主题
2. 说清这篇拆了什么
3. 给出读者收益

坏摘要常见问题：

- 太像标题重复
- 太像导语空话
- 只说主题，不说价值

## 9. 质量自检

交付前逐项检查：

1. 标题是否已经从至少 5 个候选里挑过。
2. 开头 3 段是否完成“场景、判断、承诺”。
3. 是否已经选定一种主骨架，而不是几种混写。
4. 每个小节是否都有自己的判断句。
5. 文中是否至少有一句可被引用的判断。
6. 文末是否至少给出一个可执行动作或明确落点。
7. 摘要是否经过单独优化，而不是沿用首段。
8. 封面图是否与标题和摘要方向一致。

## 10. 一套最稳的默认骨架

如果没有更好的灵感，直接用这套结构：

1. 开头：事件或触发点
2. 判断：这件事真正重要的地方
3. 拆解一：为什么会这样
4. 拆解二：这对谁影响最大
5. 拆解三：以后该怎么做
6. 结尾：回收观点 + 行动建议

这套结构适合大多数“热点分享 / 产品观察 / 创作者方法 / 嘉宾分享改写”主题。
